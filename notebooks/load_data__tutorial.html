Tutorial: Load data¶

Create a run config file.¶

The run config file is at the heart of all your data. Here, all paths file I/O, formats and meta data specific to your data are defined. It’s based on configobj (see specifications).

See configuration files for general instructions.

Download run config template

Let’s take a look at the different parts of the file.

Define run_config file header¶

You can add arbitrary header information.

experimenter = John Doe
experimenter_mail = john.doe@hts.org

plate_names defines by which names your plates are displayed and accessed:

plate_names = my_first_plate_name, my_second_plate_name, my_third_plate_name

In case base_path is defined, it is prepended to all other occurences of *path in this run config file.

base_path = /this/path/is/always/prepended

Define run_config readout data source¶

The readout data is defined with a [readout] tag. The readout data itself may have multiple levels. First, each type of readouts is given a name, e.g. [[my_readouts_name]]. For each type of readouts, there may be several readouts (for example, the raw data file may contain multiple measurements for each well).

Example:

[readout]
    [[my_readouts_name]]
        path = local/path/to/my/raw/data
        filenames = experiment 1.csv, experiment 2a.csv, experiment_2b.csv
        format = csv
        delimiter = ','

The tag format defines the format of the datafile. Currently implemented are csv, simple excel, and envision_csv.

Additional parsers can easily be added to Generic: plate_data_io or Readout only: readout_io and integrated in HTS.

One file per plate.¶

For each plate, one data file is defined. Either, they are listed:

filenames = experiment 1.csv, experiment 2a.csv, experiment_2b.csv

or, when the filenames follow a simple pattern, this can be shortened to:

filename = experiment2_{}.csv
filenumber = a, b, c

One file for all plates¶

Commonly, HTS data is saved with every sample given in one row in a .csv file. We can read this data with HTS and map it back to the plate layout with format = csv_one_well_per_row

For example:

[[my_readouts_name_3]]
    path = local/path/to/my/raw/data_3
    format = csv_one_well_per_row
    column_plate_name = plate_name  #What is the column name of the experimental plate?
    column_well = plate_well   #What is the column name of the experimental well?
    columns_readout = readout_1, readout_2, readout_3  #Which columns contain readouts?
    columns_meta = extra_info_1, extra_info_2  #Which columns contain other interesting meta information?
    width = 24  #What is the plate layout width?
    height = 16  #What is the plate layout height?

Load the data with the run config¶

In [1]:

from hts.run.run import Run
from hts.paths import TUTORIAL_DATA_NORMALIZATION_PATH

config_path = TUTORIAL_DATA_NORMALIZATION_PATH

run = Run.create(path=config_path, origin="config")

print(run)

<Run instance>
Path to run config file: /Users/elkeschaper/Python_projects/HTS/test_data/Runs/run_config_tutorial_1.txt
Number of plates: 6
width: 24
height: 16

With this run instance, we can now create a quality control report or different types of analyses.
