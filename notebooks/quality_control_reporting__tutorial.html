Tutorial: Create a Quality control report¶

HTS automatically generates Quality control reports, with a large number of diverse quality control statistics and plots currently implemented.

For reporting HTS uses the R library Knitr. Knitr allows for output as .html and .pdf (default: .html).

To comply with reproducible research standards, code, data and reports are all provided in one result folder.

Follow this tutorial to learn how to create a versatile Quality control report for your data with HTS:

Create a protocol_config file.¶

The protocoll config defines standardized procedures that are run on all of your run_config files. It is based on configobj (see specifications).

Follow the protocol_config tutorial, or download protocoll config template, or simply create an empty textfile for your protocol.

Add Quality control instructions to the protocol_config .¶

In this simple example, only two quality control plots are defined:

[My personalized Quality control report]
    tags = qc, foo, bar
    method = qc_knitr.create_report
    title = This is my favorite Quality control report
    resultfile_tag = my_quality_control
    [[1. Plate layout]]
        method = plate_layout
        text = "You may add an explanatory text here."
    [[2. My readout]]
        method = heat_map
        text = "You may add an explanatory text here."
        [[[knitr_options]]]
            options = "fig.height=7"
        [[[filter]]]
            y = my_signal
            [[[[sample_type]]]]
                values = s, pos, neg
                is_negated = False
            [[[[sample]]]]
                values = pos_1
                is_negated = True

Every quality control plot consists of at least a unique name and the method definition. An explanatory text may be defined.

[[2. My readout]]
    method = heat_map
    text = "You may add an explanatory text here."

See a [list of all currently implemented Quality controls here.

All options used in knitr chunks may be set:

[[[knitr_options]]]
    options = "fig.height=7"

Also, the quality control may be run on a filtered set of the data only: For example, to do the quality control only on positive control 0 (pos_0) and on samples, you may add:

[[[[sample_type]]]]
    values = pos, s
    is_negated = False
[[[[sample]]]]
    values = pos_1
    is_negated = True

Connect your high-throughput screening data set to the protocol_config.¶

Create a run config for your data.

First, connect the run_config to the protocol_config that contains the QC instructions by adding these lines:

# Connect the run config data to the protocol config:
[protocol]
    path = local/path/to/protocol.csv
    format = config

Then, define where the Quality control reports are saved for this data set, by adding:

[My personalized Quality control report]
    qc_result_path = QC/20160418_siRNA_BeiValeria/

* The name of the quality control tags in `procol_config` and `run_config` (Here: *My personalized Quality control report*) need to match.

Create the Quality control report: Example¶

We will use a prepared test data set.

In [3]:

# %matplotlib inline is only necessary in inline mode
%matplotlib inline
import logging
import os

from hts.data_tasks import qc_matplotlib
from hts.run.run import Run
from hts.paths import TUTORIAL_QC_PATH

logging.basicConfig(level=logging.ERROR)
config_path = TUTORIAL_QC_PATH

run = Run.create(path=config_path, origin="config")

run.qc()

{}

run.qc() invokes all protocol tasks tagged with qc. You can add an arbitrary number of tasks to the procol file, tag them and invoke them with run.do_task(tag="my_tag").
